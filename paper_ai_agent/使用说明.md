# 课程论文 AI 助手 — 使用说明

本程序是一个命令行智能体，基于**智谱 AI GLM** 大模型，可自动调用工具完成本地 PDF 论文阅读与 Google Scholar 学术文献检索，适用于课程论文写作、文献综述、论文摘要等任务。

---

## 一、功能概览

| 工具编号 | 功能 | 说明 |
|---------|------|------|
| 1 | 读取本地 PDF | 提取论文全文文字，用于摘要、分析、引用 |
| 2 | 学术文献检索 | 通过 SerpAPI 联网检索 Google Scholar，返回标题、年份、摘要等 |

模型会根据你的问题**自动决定**是否调用工具、调用几次，并在控制台打印完整的调度日志（用户输入、工具调用、工具返回、最终回答）。

---

## 二、环境要求

- **Python** 3.10 及以上
- 可访问互联网的网络环境（调用智谱 API 与 SerpAPI）
- 两个 API 密钥：
  - [智谱 AI 开放平台](https://open.bigmodel.cn/) — 大模型对话
  - [SerpAPI](https://serpapi.com/) — 学术文献检索

---

## 三、安装步骤

### 1. 进入项目目录

```bash
cd paper_ai_agent
```

### 2. 创建并激活虚拟环境（推荐）

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate
```

### 3. 安装依赖

```bash
pip install -r requirements.txt
```

### 4. 配置环境变量

将模板复制为 `.env` 并填写密钥：

```bash
# Windows
copy .env.example .env

# macOS / Linux
cp .env.example .env
```

编辑 `.env` 文件：

```env
ZHIPU_API_KEY=你的智谱密钥
SERPAPI_API_KEY=你的SerpAPI密钥
ZHIPU_MODEL=glm-4-flash
```

### 5. 准备 PDF 论文（可选）

将待分析的 PDF 文件放入 `papers` 目录（若目录不存在，请手动创建）：

```
paper_ai_agent/
  papers/
    sample.pdf
    某篇论文.pdf
```

也可在提问时使用 PDF 的绝对路径。

## 四、启动方式

程序支持三种启动方式，控制台选项均为**数字**。

### 方式 1：数字菜单（无参数启动）

```bash
python main.py
```

启动后按提示选择：

```
  1 — 单次提问
  2 — 交互对话
  0 — 退出
```

### 方式 2：单次提问

直接在命令行传入问题：

```bash
python main.py "请阅读 sample.pdf 并总结核心观点"
```

```bash
python main.py "检索 2020 年以来关于 Transformer 注意力机制的代表性论文"
```

### 方式 3：交互对话

```bash
python main.py --interactive
# 或
python main.py -i
```

进入交互模式后可持续多轮提问；输入 **0** 退出。

### 查看帮助

```bash
python main.py -h
```

## 五、使用示例

### 示例 1：阅读并总结本地论文

1. 将 PDF 放入 `papers/` 目录，例如 `papers/attention.pdf`
2. 运行：

```bash
python main.py "请阅读 attention.pdf，用 300 字总结其主要贡献"
```

模型会自动调用工具 1 读取 PDF，再基于原文生成摘要。

### 示例 2：检索外部文献

```bash
python main.py "帮我找 5 篇 2018–2024 年关于 BERT 预训练的代表性论文，列出标题和年份"
```

模型会自动调用工具 2 进行学术检索。

### 示例 3：结合本地论文与外部文献

```bash
python main.py "先阅读 my_paper.pdf，再检索与其主题相关的近五年文献，对比异同"
```

模型可能依次调用工具 1 和工具 2，多轮协作完成任务。

### 示例 4：交互模式连续提问

```bash
python main.py -i
```

```
你：请总结 papers/review.pdf 的研究方法
你：再帮我补充 3 篇相关参考文献
你：0
```

## 六、配置说明

所有配置通过项目根目录下的 `.env` 文件管理，常用项如下：

| 变量名 | 是否必填 | 默认值 | 说明 |
|--------|---------|--------|------|
| `ZHIPU_API_KEY` | 是 | — | 智谱 AI 接口密钥 |
| `SERPAPI_API_KEY` | 是 | — | SerpAPI 接口密钥 |
| `ZHIPU_MODEL` | 否 | `glm-4-flash` | GLM 模型名称 |
| `ZHIPU_BASE_URL` | 否 | 智谱官方地址 | 一般无需修改 |
| `PDF_BASE_DIR` | 否 | `./papers` | 本地 PDF 默认存放目录 |
| `MAX_TOOL_ROUNDS` | 否 | `8` | 单次对话最多工具调用轮次 |
| `SERPAPI_NUM_RESULTS` | 否 | `5` | 单次检索默认返回条数 |

### 常用 GLM 模型

| 模型 | 特点 |
|------|------|
| `glm-4-flash` | 速度快、成本低，适合日常使用（默认） |
| `glm-4-air` | 速度与效果均衡 |
| `glm-4-plus` | 能力更强，适合复杂推理 |

在 `.env` 中修改 `ZHIPU_MODEL` 即可切换。

## 八、自定义系统提示词

编辑 `prompt.txt` 可调整助手的行为风格、回答格式与工具使用策略。修改后重新启动程序即可生效。

当前默认要求：

- 查找外部文献时调用工具 2
- 回答使用中文，结构清晰

## 十、常见问题

### 1. 启动失败：缺少必要环境变量

**原因**：`.env` 未配置或密钥为空。

**解决**：确认已复制 `.env.example` 为 `.env`，并正确填写 `ZHIPU_API_KEY` 与 `SERPAPI_API_KEY`。

### 2. 文件不存在

**原因**：PDF 路径错误或文件未放入 `papers` 目录。

**解决**：

- 使用相对于 `papers/` 的文件名，如 `sample.pdf`
- 或使用绝对路径，如 `E:\docs\paper.pdf`

### 3. PDF 未能提取到文字

**原因**：PDF 为扫描件或图片型，无法提取可选文字。

**解决**：使用 OCR 工具先将 PDF 转为可搜索文本，或换用文字版 PDF。

### 4. 学术检索失败或结果为空

**原因**：SerpAPI 密钥无效、网络问题，或检索词过于狭窄。

**解决**：检查 `SERPAPI_API_KEY`、网络连接，并尝试更通用的英文检索词。

### 5. 已达到最大工具调用轮次

**原因**：任务过于复杂，超过 `MAX_TOOL_ROUNDS`（默认 8 轮）。

**解决**：将问题拆分为多个小任务，或在 `.env` 中适当增大 `MAX_TOOL_ROUNDS`。

### 6. 终端出现「意外的标记 PS」或「无法将 File 项识别为 cmdlet」

**原因**：把整段终端输出（含 `(base) PS E:\...>` 提示符和 Python 报错）复制粘贴回 PowerShell 执行了。PowerShell 会把这些内容当作命令解析，从而报错。

**解决**：只输入命令本身，不要粘贴提示符和报错文本。正确示例：

```bash
python main.py "请阅读 PIAFusion.pdf，用 300 字总结其主要贡献"
```

若刚才误粘贴导致终端状态异常，按 `Ctrl+C` 取消当前输入，重新输入上述命令即可。

### 7. 智谱 AI 认证失败（401 / 令牌已过期或验证不正确）

**原因**：`.env` 中的 `ZHIPU_API_KEY` 无效、已过期或被撤销。

**解决**：

1. 登录 [智谱 AI 开放平台](https://open.bigmodel.cn/)
2. 在「API 密钥」页面新建或重新生成密钥
3. 将新密钥写入 `.env` 的 `ZHIPU_API_KEY=` 后保存
4. 重新运行程序

注意：密钥格式通常为 `xxxxxxxx.xxxxxxxx`，等号后不要加引号或多余空格。

## 十一、注意事项

1. **密钥安全**：请勿将 `.env` 文件提交到公开仓库或分享给他人。
2. **API 费用**：智谱 AI 与 SerpAPI 均为按量计费服务，使用前请了解各自定价。
3. **学术规范**：AI 生成的内容仅供参考，正式提交论文前请自行核实引用与观点。
4. **扫描版 PDF**：本程序使用文字提取，不支持 OCR，扫描件可能无法读取。
